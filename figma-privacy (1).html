Lace - Privacy Policy

Effective Date: December 2025
Last Updated: December 2025

1. Overview

Lace ("we", "our", or "us") provides a version control and collaboration workflow for designers using Figma. This policy explains how we access, process, and protect information when you use the Lace plugin.

2. Information We Collect

2.1 Figma Data

When you authorize Lace through Figma OAuth, we may access:

  • Frame and File Content: Design structure, text, layers, and component metadata used to create version summaries.
  • File Metadata: File name, file key, and timestamps used to organize projects and versions.
  • Comments: Comments and replies from the file to display context and enable bi-directional communication.
  • User Information: Your Figma display name and user ID to attribute activity correctly.

2.2 Authentication Data

  • Encrypted OAuth access tokens
  • Encrypted OAuth refresh tokens
  • Plugin session authentication tokens

3. How We Use Your Information

We use collected information solely to:

  • Generate and display version history for selected frames.
  • Create automated summaries of design changes.
  • Sync comments and replies between Lace and Figma.
  • Identify which user authored or updated a version.
  • Maintain an authenticated connection with Figma.

4. Data Storage and Security

4.1 Storage

  • Version metadata, activity logs, and summaries may be stored securely on Lace servers.
  • OAuth credentials are encrypted using industry-standard security practices.
  • We store only the minimum data necessary to operate the plugin.

4.2 Security Measures

  • All network communication is encrypted via HTTPS.
  • Access tokens are encrypted and never shared with other users.
  • Only authorized services can access stored data.

5. Data Sharing

We do not:

  • Sell or monetize user data.
  • Share design files, metadata, or comments with third parties.
  • Expose your Figma content outside Lace workspaces.

6. Data Retention

  • Stored versions and summaries remain available while your Lace account is active.
  • OAuth tokens are deleted immediately when you disconnect Lace from Figma.
  • You may request deletion of stored content at any time.

7. Your Rights

You may:

  • Disconnect Lace from Figma at any time.
  • Request access to stored account-related data.
  • Request deletion of stored information.

8. Scope of Figma Access

Lace currently requests only the following Figma scopes:

  • current_user:read
  • file_comments:read
  • file_comments:write
  • file_content:read
  • file_metadata:read
  • file_versions:read

We do not request enterprise design system scopes. If future features require additional permissions, we will notify users before requesting them.

9. AI Processing and Third-Party Sharing (OpenAI)

Lace uses OpenAI’s API to generate automated summaries, detect changes, and enhance version review features. To provide these capabilities, Lace may send the following information from the selected Figma file to OpenAI:

  • Text content contained in design frames
  • Frame preview images and rendered design snapshots
  • Figma comments, replies, and discussion context
  • Layer names, structure, and component metadata
  • Version comparison results and change descriptions

Data shared with OpenAI is used solely to generate results requested by the user. Outputs may be stored as part of version history within Lace. We do not allow OpenAI to use this data to train models. All processing is governed by OpenAI’s API Terms and Privacy Policy.

Because AI processing is required for core functionality, Lace does not offer an opt-out from OpenAI processing at this time.
